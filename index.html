Filter Basics

Files Used

There are 3 files the lootfilter will need for proper operation and user control.

If any of these 3 are out-of-date or corrupted/bugged, then functionality will be negatively impacted


D2RHUD.DLL - This file is used to interact with the game directly. It acts as the middle-man between LootFilter.lua and game functionality.
Location: D2RLAN/Launcher/D2RHUD.DLL
Latest Version: 1.4.2
Code Source: https://github.com/locbones/D2RHUD-2.4

LootFilter.LUA - This file acts as the control logic on the filter side, acting as the middle-man between LootFilter_Config.lua and D2RHUD.dll.
Location: D2RLAN/D2R/LootFilter.LUA
Latest Version: 1.3.6
Code Source: https://github.com/locbones/D2RLAN-Filters

LootFilter_Config.LUA - This file is what's ultimately controlled by YOU. Add rulesets that will actively engage with the game in real-time.
Location: D2RLAN/D2R/LootFilter_Config.LUA
Latest Version: 1.0.0
Code Source: https://github.com/locbones/D2RLAN-Filters


Data Types

This loot filter allows you to setup 'active' rulesets to precisely control what items you see, and how you see them.

They are active, because they will interact with the game directly, and allow you to process rules based on item attributes (quality, stats, ilvl, etc).

Items can be matched by multiple rules, allowing a variety of separate modifications to be done on a broader basis.


All controls are broadly split into three different types: Conditions, Displays and Global Options

- Conditions control what the rule must satisfy in order to be 'matched' for that item's Display option(s)

- Displays control what happens when all of the conditions are satisfied. Such as hiding the item, altering it's color, adding text, etc.

- Global Options control filter-wide logic, such as desired language, reload message text, filter level, debug mode, and more.



Conditions

Core Conditions

These are conditions that will help you determine what the item is; it's core identity

Item Code(s)

code/codes - Defines the itemcodes that should match this condition. Retrieved from Armor, Misc, or Weapons.txt code column

  • If no other conditions are set, it will match all items of that code or codes.
  • A special case exists if using codes: “allitems” to match all items.

Usage Example(s)

code = “abc”

OR

codes = { “abc”, “def”, “ghi” }

OR

codes = “allitems”

Item Level

ilvl - Defines the item level(s) that should match the condition. Uses a numeric value


Usage Example(s)

ilvl = 55

OR

ilvl = "55+"
Set/Unique Identifier

index - Defines the set or unique ID# that should match the condition. Retrieved from Set/UniqueItems.txt *ID column*

  • *ID# is technically hardcoded to the row in the .txt file, but the *ID column will usually match it
  • Set and Unique items will have a different index than each other, as they read from different files
  • This feature can be used for hosts or single-player games to determine the exact set/unique before it's identified

Usage Example(s)

index = 122

OR

index = { 122, 123, 124 }
Quality

quality - Defines the quality that should match the condition. Uses a numeric value 1-9 to determine quality

ValueQuality
1Low
2Normal
3Superior
4Magic
5Set
6Rare
7Unique
8Crafted
9Tempered

Usage Example(s)

quality = 3

OR

quality = { “1”, “3”, “5” }

OR

quality = “3-”

Rarity

rarity - Defines the rarity that should match the condition. Uses a numeric value 1-3 to determine rarity

ValueRarity

3Elite

Usage Example(s)

rarity = 3

OR

OR

Subjective Conditions

These are conditions that will help you determine why you care about the item; subjective to what you desire

Sockets

sockets - Defines the socket count that should match the condition. Uses a numeric value 0-6 to determine sockets


Usage Example(s)

sockets = 3

OR

sockets = { “1”, “3” }

OR

sockets = “3-”

Ethereal

ethereal - Defines the ethereal status that should match the condition. Uses either true or false


Usage Example(s)

ethereal = true
Runeword

runeword - Defines the runeword status that should match the condition. Uses either true or false


Usage Example(s)

runeword = true
Identified

identified - Defines the identification status that should match the condition. Uses either true or false


Usage Example(s)

identified = true
Max Sockets

maxsock - Determines if the item is at the current max socket count for it's item level. Uses either true or false

  • If an item is currently level 30, and has 3/3 sockets, it will return true - even if at item level 40 it unlocks 4+ sockets

Usage Example(s)

maxsock = true

Situational Conditions

These are conditions that will help you determine when you care about the item; situational to what you need

Filter Level/Strictness

filter_levels - Defines the filter_level this rule should be assigned to. Uses user-defined values specified in Global Variables

  • filter_levels are optional; if none are specified, default values will be assumed
  • filter_levels can either be treated as numeric values or titles/text

Usage Example(s)

filter_levels = 1

OR

filter_levels = { “1”, "2" }

OR

filter_levels = { “Early-Game”, "Mid-Game", "End-Game" }
Location

location - Defines the location (literal) that should match the condition. Uses a pre-defined value


LocationDescription
ongroundThis is default behavior if no location is specified
onplayerIn the player's inventory, cube or personal stash
equippedActively equipped by the player
atvendorA 'loose' reference to any item that doesn't match above conditions

Usage Example(s)

location = "onground"

OR

location = { “onground”, “onplayer”, "equipped" }
Area

area - Defines the location (level) that should match the condition. Uses a user-configured value specified in LootFilter.lua using the AreaID Table Map

AreaIDs = { ["Rogue Encampment"] = 1, ["Blood Moor"] = 2, etc }
  • If you are not making a loot filter for ReMoDDeD, then you must comment out or delete Line 39: mod = "RMD"
  • If you have added new levels or edited names of existing ones, you will need to update this Table Map yourself

Usage Example(s)

area = "Blood Moor"

OR

area = { "Blood Moor", "Cold Plains", "Stony Field" }
Difficulty

difficulty - Defines the difficulty that should match the condition. Uses a pre-defined value


Difficulty
Normal
Nightmare
Hell

Usage Example(s)

difficulty = "Normal"

OR

difficulty = { “Nightmare”, “Hell” }
Stat / Pstat

stat / pstat - Defines the stats/values that should exist to match the condition. Uses special and varying syntax

  • Stat is used to check stats on an item. Pstat is used to check stats on the player

  • Due to how D2 processes stats, some will require param values, while others will not [More Info]
  • This condition will also support comparison Operands to check value ranges
  • This condition will require the Stat ID as the 'index' to determine which stat to check

Usage Example(s)

stat = { index = 93, op = "<=", value = 30 }    --Check if 'item_fasterattackrate' is less than or equal to 30

OR

stat = { index = {0, 1, 2, 3}, op = "==", value = 15 }    --Check if 'strength', 'dexterity', 'vitality' and 'energy' are equal to 15

OR

pstat = { index = 12, op = "<=", value = 25 }    --Check if player is 'level' 25 or less

OR

stat = { index = 97, op = ">=", param = 38, value = 1 }    --Check if item grants +1 to Charged Bolt (or higher)


Displays

Item Name

These are displays that will help you alter how the item name is ultimately shown

Name Override

name_override - Defines the new name the item should use, instead of the one specified from the string files


Usage Example(s)

name_override = “SoJ”

prefix - Defines the text that should be added before the item name


Usage Example(s)

prefix = “[Perfect] ”
Name Suffix

suffix - Defines the text that should be added after the item name


Usage Example(s)

suffix = “[Eth]”
Name Style

name_style - Defines the color style that should be applied to the text. Uses pre-defined options

  • Static versions of all styles also exist, that do not contain animated coloring (add ' Static' at end)
StyleExample

Rainbow Group
Cotton Candy
Frost Wave

name_style = “Frost Wave”

Description/Tooltip

These are displays that will help you alter how the item description is ultimately shown

Description Prefix

prefix_desc - Defines the text that should be added before the item description


Usage Example(s)

prefix_desc = “This item is cool because: ”
Description Suffix

suffix_desc - Defines the text that should be added after the item description


Usage Example(s)

suffix_desc = “Here's some more info: ”
Tooltip Background

background - Defines the tooltip background color for the item

  • Uses a series of 4 numeric values to determine the RGB color
  • The first 4 values are for R, G, B, A respectively

Usage Example(s)

background = { 110, 110, 255, 255 }
Tooltip Style

background_style - Defines the color style that should be applied to the text tooltip/background. Uses pre-defined options

StyleExample
Rainbow
Ocean Drift
Fading Ember
Midnight Memory
Verdant Bloom
Sapphire Sonnet
Tamarillo Glow

Usage Example(s)

background_style = “Fading Ember”
Tooltip Border

border - Defines the tooltip border's color and width for the item

  • Uses a series of 5 numeric values to determine the RGB color and the width
  • The first 4 values are for R, G, B, A respectively. The 5th value is the border pixel size

Usage Example(s)

border = { 110, 110, 255, 255, 5 }

Actions

These are displays that will help perform an action, such as hiding an item, or displaying a notification

Hide

hide - Hides both the item text and the world graphic for the item. Uses a true or false value

  • If no hide value is specified, it is assumed false
  • The item is not deleted/removed permanently, it removes it from the active item table
  • To restore the visual of a hidden item, undo your affected rule, then visit a new area or game, and reload your filter

Usage Example(s)

hide = true

Notifications

notify - Displays a chat message when the item has matched the conditions. Uses a text value

  • Notify can also use the {link} option in Output Values

Usage Example(s)

notify = "Woohoo! {link}"


Global Options

Output Values

The following arguments can be used to output the specified value to the item name, description, or chat notification

Outputs

{code} - Adds the code value to the output

{ilvl} - Adds the ilvl value to the output

{rarity} - Adds the rarity value to the output

{quality} - Adds the quality value to the output

{name} - Adds the name text to the output

{ethereal} - Adds the ethereal status to the output

{sockets} - Adds the socket count to the output

{index} - Adds the ID value to the output

{itype} - Adds the itype value to the output

{maxsock} - Adds the maxsock status to the output

{stat=(xxx)} - Adds the stat / pstat value to the output


Available Colors
  • All text following the color code will be changed to the specified color, including lines above or below due to how D2 handles tooltips
  • You will sometimes need to 'close' the color by adding the 'default' color (such as blue) at the end of the text

{white} - Changes the color to White (Uses ÿc0)

{red} - Changes the color to Red (Uses ÿc1)

{green} - Changes the color to Green (Uses ÿc2)

{blue} - Changes the color to Blue (Uses ÿc3)

{gold} - Changes the color to Gold (Uses ÿc4)

{grey/gray} - Changes the color to Grey (Uses ÿc5)

{black} - Changes the color to Black (Uses ÿc6)

{tan} - Changes the color to Tan (Uses ÿc7)

{orange} - Changes the color to Orange (Uses ÿc8)

{yellow} - Changes the color to Yellow (Uses ÿc9)

{purple} - Changes the color to Purple (Uses ÿc;)

{dark green} - Changes the color to Dark Green (Uses ÿcA)

{turquoise} - Changes the color to Turquoise (Uses ÿcN)

{pink} - Changes the color to Pink (Uses ÿcO)

{lilac} - Changes the color to Lilac (Uses ÿcP)


Operands

The following arguments can be used to tell certain conditions to use logic such as NOT and OR

NOT

NOT - Allows you to exclude all of the entries for a specific condition


Usage Example(s)

area = NOT {"Rogue Encampment", "Lut Gholein", "Kurast Docktown", "The Pandemonium Fortress", "Harrogath"}

OR

codes = NOT { "abc", "def", "ghi" }

OR

OR - Allows you to choose between 1 or more entries for a specific condition


Usage Example(s)

stat = OR { index = {0, 1}, op = "==", value = 15 }

Operands

< - Value is less than

- - Value is less than (Example: 3-)

> - Value is greater than

+ - Value is greater than(Example: 3+)

== - Value is equal to

<= - Value is less than or equal to

>= - Value is greater than or equal to

between - Value is between x and y (used on Stat/Pstat condition only)


Usage Example(s)

stat = { index = 12, op = "==", value = 15 }

OR

sockets = "3+"

OR

stat = { index = 12, op = "between", value = { 15, 25 } }


Filter Controls

The following arguments can be applied as a global modifier for your loot filter, changing it's overall behavior or output.

  • The options listed below must be applied before your 'rules' section, as shown in the Barebones Template
Reload Message

reload - Allows you to specify the chat message displayed when config is reloaded in-game


Usage Example(s)

reload = "My Loot Filter {red}v1.0"
Debug Mode

debug - Allows you to toggle the Debug Mode, which will display rule match logic in chat


Usage Example(s)

debug = true
Filter Overrides

allowOverrides - Allows other loot filters, such as a mod's default filter, to be loaded alongside your own


Usage Example(s)

allowOverrides = true

Filter Level

filter_level - Displays/Controls your currently active filter level. Controllable via D2RLAN hotkeys in-game.


Usage Example(s)

filter_level = 1

Filter Titles

filter_titles - Allows you set how filter_levels are displayed in-game, using either numbers or text


Usage Example(s)

filter_titles = { "1", "2", "3" }

OR

filter_titles = { "Early-Game", "Mid-Game", "End-Game" }

Filter Language

language - Allows the loot filter to change text output depending on language selected. English (enUS) is defaulted


Usage Example(s)

language = "enUS"


Demo Templates

Barebones Template

This template will show you how the filter should be structured. No rules will be included in this template

All of the global variables are optional, they will be included for clarity sake only


--- Filter Title: My Filter
--- Filter Type: (None)
--- Filter Description: This is my blank filter
--- Filter Link: https://mylink.con/myconfig.lua
return {
    reload = "My Loot Filter {red}v1.0",
    allowOverrides = true,
    language = "enUS",
    filter_level = 1,
    filter_titles = { "Early-Game", "Mid-Game", "Late-Game" },
    debug = false,
    rules = {
    -- Rule #1
        {
            codes = "allitems"
            prefix = "{red}[Filter Working]\n"
        },
        -- Rule #2
        {
            codes = "allitems"
            suffix = "\n{purple}[Still Working]"
        }
    }
}
